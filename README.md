# AndroidApp


As the Project Manager and co-author of the "Gluko" mobile application, developed at the Lodz University of Technology within the Biomedical Engineering program, I led the development of a specialized tool for monitoring blood glucose levels. My primary objective was to design a system that seamlessly integrates medical data with the user's daily nutritional habits. I focused on ensuring a precise correlation between glucose measurements and meal intake, which is fundamental for effective self-management of diabetes and insulin resistance.

In terms of functionality, I oversaw the implementation of modules for manual data entry, including exact measurement times, glucose values, and detailed meal descriptions with carbohydrate content. As the team leader, I supervised the user interface design to ensure that data—presented in weekly, monthly, and yearly views—remains clear and actionable. This allows users to easily identify trends and recognize which specific food products or times of day cause undesirable glucose spikes, directly facilitating dietary optimization.

From a technical standpoint, I managed the application build process within the Android environment using the Gradle standard, which allowed for efficient dependency management and build automation. I configured the project structure using Kotlin Script (KTS), ensuring the integrity of configuration files such as build.gradle.kts and settings.gradle.kts. Utilizing modern development tools enabled our team to maintain high code quality and ensure full compatibility with AndroidX libraries, which is critical for the stability of a medical application.

During the production process, I placed heavy emphasis on maintaining repository order and local configuration security. I supervised the correct implementation of files like .gitignore and local.properties to ensure that sensitive local data, such as SDK paths, were not committed to the version control system. By implementing gradlew scripts, I guaranteed the reproducibility of the build process across different workstations, which was essential for effective team collaboration and meeting project milestones.

Although the current version of the "Gluko" app successfully achieves its goals in glycemic monitoring, as the Project Manager, I see significant potential for further technological evolution. In the future, I plan to expand the system with automated data analysis modules using machine learning algorithms and integration with external Continuous Glucose Monitor (CGM) sensors. The solution we have created serves as a robust engineering foundation for a modern telemedicine tool that significantly improves patient quality of life through enhanced health awareness.
